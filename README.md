# Yang Lintao Project

一个基于纯前端技术的浏览器视频拼接工具，所有处理在本地完成，无需上传至服务器，保护用户隐私。

## 项目预览

打开 `index.html` 即可使用，无需安装任何软件或搭建服务器。

## 功能特性

- **本地上传**：支持从本地选择两个视频文件，支持点击选择和拖拽上传
- **视频预览**：上传后可直接在页面内播放预览
- **顺序调整**：支持一键交换两个视频的前后拼接顺序
- **视频合并**：将两个视频首尾拼接为一个完整视频
- **进度显示**：合并过程中实时显示处理进度
- **预览下载**：合并完成后可预览效果并一键下载
- **一键重置**：清空所有内容，重新开始操作

## 技术栈

| 技术 | 用途 |
|------|------|
| HTML5 | 页面结构 |
| CSS3 | 界面样式、渐变背景、响应式布局 |
| JavaScript (ES6+) | 核心交互逻辑 |
| Canvas API | 视频帧绘制与画面合成 |
| MediaRecorder API | 视频流录制与编码 |
| Web Audio API | 音频流捕获与合并 |
| Blob / URL API | 文件生成与下载 |

## 技术实现原理

### 整体流程

```
用户上传视频 → 解析视频元数据 → Canvas逐帧绘制 → MediaRecorder录制 → 生成Blob → 下载
```

### 详细说明

1. **视频解析**：通过 `<video>` 元素加载用户上传的文件，读取分辨率、时长等元数据
2. **画布合成**：创建 `<canvas>` 画布，取两个视频中较大的分辨率作为输出尺寸
3. **逐帧绘制**：依次播放两个视频，使用 `requestAnimationFrame` 将每一帧绘制到 Canvas 上
4. **音频捕获**：通过 Web Audio API 的 `createMediaElementSource` 捕获视频音频流
5. **流录制**：使用 `canvas.captureStream()` 捕获画面流，结合音频流，通过 `MediaRecorder` 录制
6. **文件生成**：录制完成后将数据块合并为 Blob 对象，生成可下载的视频文件

### 关键技术点

- **自适应缩放**：不同分辨率的视频会自动等比缩放并居中显示，避免画面变形
- **编码格式选择**：优先使用 VP9 编码（更好的压缩率），自动降级到 VP8
- **纯客户端处理**：完全在浏览器中运行，不依赖任何后端服务

## 使用方法

1. 用浏览器（推荐 Chrome）打开 `index.html`
2. 点击左侧区域上传第一个视频（前半段）
3. 点击右侧区域上传第二个视频（后半段）
4. 如需调整顺序，点击中间的 ⇄ 按钮
5. 点击「合并视频」按钮，等待处理完成
6. 预览合并效果，点击「下载合并视频」保存

|--------|---------|

| Firefox 80+ | 支持 |
| Safari 14+ | 部分支持 |

## 已知限制

- 输出格式为 WebM（浏览器原生支持的编码格式）
- 视频越长，处理时间越长（实时速率录制）
- 受浏览器内存限制，建议单个视频不超过 500MB
